First-Class Types

First-Class Types are entities that can be passed as a parameter, returned from a subroutine, or assigned to a variable

undefined
value of any variable that does not exist
null
value of any uninitialized variable
boolean
used to record a value of either true or false
number
holds any type of number, either an integer or a real number, values are immutable
string
stores a string of characters, typically making up either a word or sentence. string variables can also be assigned empty strings.
object
data structure containing mutable key-value pairs
function
used to organize functionality into clean, reusable modules that can be accessed from any other JavaScript to perform specific tasks.

Operators


            * / % (binary)
  • coerce both Number and operate
            + (binary)
  • Add two Numbers together
  • If one side is a String, convert other to a String and concatenate
            - (unary)
  • coerce to Number and negate

Operators


              = (binary)
              // Assignment
              var a = "whatever dude";
            
              == (binary)
              // Equality
              if(a == 5) {...}
              if("1" == 1) {...} // true
            
              === (binary)
              // Equality without type coercion
              if(a === 5) {...}
              if("1" === 1) {...} // false

Using === is faster than using ==, so use === whenever possible

Scoping


Variables are created on the fly:
              b = 1 + 2

Using var creates the variable at the innermost scope:
              var b = 1 + 2;
              var a = 200;
              if(typeof a != "undefined") {
                var b = String(a);
                assertEqual(typeof b, "string");
                assertEqual(b, String(b));
                assertEqual(b, b.toString());
              }
              // b == "200" out here

Braces DO NOT create a new inner scope
ONLY functions create a new inner scope

Scoping

              var b = 1 + 2;
              assertEqual(window.b, 3);
              
              function mytest() {
                if(true) {
                  var a = "str";
                  assertUndefined(window.a); // true
                  assertEqual(a, "str");     // true
                }
              }
              mytest();
              assertUndefined(window.a);     // true

In browser, all variables in the global scope are on the window object

DOM (Document Object Model)

document
Top-most element of a DOM, tree-structured
Contains the root node where everything starts
Used to create other elements
Used to scan for elements
Node
Text Node, Comment Node, Element Node, Attribute Node
Element
Created by every opening tag in an HTML document
May contain other elements as child nodes
May contain text nodes as child nodes
May have sibling elements

DOM — document

document.getElementById("id-tag-value")
returns first Element with specified id attribute otherwise null
document.getElementsByClassName("class-tag-value")
returns list of Element with specified class attribute
document.createElement("div")
returns a new "div" Element with no children and no parent
document.createTextNode("some HTML unescaped text")
returns a text Node that can be added to an Element
document.createDocumentFragment()
returns a fragment that can be added to a document

More at https://developer.mozilla.org/en/DOM/document

DOM — Node Functions

appendChild(childNode)
adds aNode as a child
insertBefore(aNode, beforeNode)
inserts aNode as a child and placed before beforeNode if beforeNode is a child of this Node, otherwise, append aNode to the end
removeChild(aNode)
removes aNode if it is a direct child
replaceChild(newChild, oldChild)
removes oldChild if it exists and replaces it with newChild
hasChildNodes()
whether this node has children
hasAttributes()
whether this node has attributes

More at https://developer.mozilla.org/en/DOM/node

textContent

innerHTML

className
